TuneBox

Music chat room using MEAN stack and YouTube API

Overview

School project for Software Start-up course. The chat room software concentrates on music streaming and connects music enthusiasts with similar interests. Artists, record labels etc can also use TuneBox for promoting their material and expanding their listener base.

Features

The users need to register and there are two types of accounts: free (with limited number of private chat rooms and concurrent listeners/room) and premium (with higher limits and ability to also create public rooms). The front page supports showing featured public rooms (B2B) and advertisements for free users.

Users can search for music videos asynchronously (displays suggestions while typing) and the results are ordered as they would be in YouTube. While creating a chat room, users can add keywords as tags, upload and crop a picture for the room and write name and description. TuneBox synchronizes the current music video between the users and also displays chat room suggestions based on the user's preferred tags.

Upcoming and Other Planned Features

  • Playlist support
  • Voting mechanism ("like" button/vote to skip a song)
  • Streaming live with camera input
  • Location based suggestions
  • One of the course's goals was to come up with a business plan, so we also designed some ideas for various fictional billing mechanisms, e.g. monthly premium subscription and buying booster packs to raise the room or listener limits.

Technical Details

The project is implemented using MEAN stack, which consists of MongoDB (NoSQL database), Express.js (web application framework), Angular.js (JavaScript MVC framework) and Node.js (the execution environment). TuneBox has a browser client and uses YouTube API as the video player and to search for videos. Additionally, we deployed a live test version in Heroku during the course.
